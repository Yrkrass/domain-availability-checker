# Domain Availability Checker

Веб-застосунок для моніторингу доступності доменів з адмін-панеллю на базі Filament.

## Можливості

- Реєстрація та авторизація користувачів
- Управління доменами (додавання, редагування, видалення, імпорт списком)
- Налаштування перевірок (інтервал, таймаут, метод HTTP)
- Автоматичні та ручні перевірки доменів
- Асинхронна обробка через чергу задач (Redis)
- Логування результатів перевірок
- Telegram-нотифікації після завершення циклу перевірок
- Дашборд зі статистикою та графіками

## Адмін-панель

### Дашборд
- Статистика: загальна кількість доменів, доступні/недоступні, активні налаштування, середній час відповіді, загальна кількість перевірок
- Графік доступності доменів за останні 7 днів (стовпчикова діаграма)
- Графік відсотку доступності за останні 30 днів (лінійний графік)

### Домени
- Додавання, редагування та видалення доменів
- **Імпорт списком** — вставити кілька доменів одразу (по одному на рядок) через зручне текстове поле з валідацією URL
- Кожен користувач бачить тільки свої домени
- Відображення статусу доступності та часу останньої перевірки

### Налаштування перевірок
- Список налаштувань з фільтрами по режиму, статусу та стану виконання
- **View-сторінка** з кнопками керування та таблицею логів яка оновлюється кожні 3 секунди в реальному часі
- Кнопка **Run** — запускає ручну перевірку (доступна тільки в режимі Manual)
- Кнопка **Stop** — примусово зупиняє поточний цикл перевірок
- Кнопки автоматично перемикаються залежно від стану виконання

### Логи перевірок
- Відображаються прямо на сторінці налаштування у вигляді RelationManager
- Автооновлення кожні 3 секунди — видно як результати з'являються в реальному часі
- Колонки: домен, доступність, код відповіді (з кольоровими badge), час відповіді, помилка
- Довгі тексти помилок обрізаються з посиланням — клік відкриває повний текст у модальному вікні
- Також доступні логи на сторінці конкретного домену

## Технології

- **PHP 8.3** / **Laravel 11**
- **Filament 3** — адмін-панель
- **MySQL** — база даних
- **Redis** — черга задач
- **Docker** — контейнеризація
- **Nginx** — веб-сервер

## Встановлення

### Вимоги

- Docker
- Docker Compose

### Кроки

1. Клонуй репозиторій:
```bash
git clone <repo-url>
cd domain-availability
```

2. Скопіюй `.env` файл:
```bash
cp src/.env.example src/.env
```

3. Налаштуй змінні в `.env`:
```env
DB_CONNECTION=mysql
DB_HOST=db
DB_PORT=3306
DB_DATABASE=domain-availability
DB_USERNAME=da_user
DB_PASSWORD=da_password

REDIS_HOST=redis
REDIS_PASSWORD=your_redis_password

TELEGRAM_BOT_TOKEN=your_bot_token
```

4. Збери і запусти контейнери:
```bash
cd docker
docker compose up -d --build
```

5. Запусти міграції та сідери:
```bash
docker exec -it domain-availability-app-1 php artisan migrate --seed
```

6. Відкрий браузер: [http://domain-availability.loc/admin](http://domain-availability.loc/admin)

## Архітектура перевірок

Перевірки працюють асинхронно через чергу задач:

1. `CheckDomainsJob` — диспатчить окремі джоби для кожного домену
2. `CheckDomainJob` — перевіряє один домен (HEAD → GET fallback)
3. Результат записується в `check_logs`
4. Після завершення циклу відправляється Telegram-нотифікація

### Методи перевірки

- **HEAD** — швидка перевірка без тіла відповіді
- **GET** — повна перевірка
- **HEAD → GET** — спочатку HEAD, якщо 405 — GET

### Режими перевірки

- **Manual** — запуск вручну через кнопку в адмін-панелі
- **Auto** — автоматичний запуск по розкладу з вказаним інтервалом

## Налаштування Telegram

1. Створи бота через [@BotFather](https://t.me/BotFather)
2. Додай токен в `.env`: `TELEGRAM_BOT_TOKEN=your_token`
3. Дізнайся свій Chat ID через [@userinfobot](https://t.me/userinfobot)
4. Вкажи Chat ID в профілі користувача в адмін-панелі

## Розгортання без Docker

1. Встанови залежності:
```bash
composer install
npm install && npm run build
```

2. Налаштуй `.env` під своє середовище

3. Додай в crontab:
```bash
* * * * * cd /path/to/project && php artisan schedule:run >> /dev/null 2>&1
```

4. Запусти воркер:
```bash
php artisan queue:work
```

## Корисні команди

```bash
# Запустити заплановані перевірки вручну
php artisan checks:run-scheduled

# Очистити чергу
php artisan queue:clear

# Переглянути провалені задачі
php artisan queue:failed
```
